# E Relations

EPC工程の依存関係をGUIで編集するローカル実行可能なMVPです。

## 構成

```txt
web/      ブラウザ画面
api/      APIハンドラと保存ロジック
server/   ローカル開発サーバー
data/     ローカルJSON保存先
docs/     GitHub Pages公開用ファイルと設計資料
```

本番想定では `web/` を Azure Static Web Apps、`api/` を Azure Functions、保存先を Azure Blob Storage に置き換えます。
ローカルではAzureを使わず、`data/plans/*.json` に保存します。

## 機能要件

機能要件は [docs/functional-requirements.md](docs/functional-requirements.md) に整理しています。
添付仕様のRelation Tool v2.1を参考にしつつ、初期版は軽く動くMVPとして次を優先します。

- タスクの追加、編集、ドラッグ配置
- FS / SS / FF / SF とラグ付き依存線の作成
- 循環依存の防止
- CPM計算、クリティカル、余裕日数の表示
- ネットワーク図、ガント、表ビュー
- 領域フィルター、検索、フォーカス一覧
- ステータス、予定日、実績日、レベル、説明メモ
- 所属/領域マスターの追加
- 依存線ラベルの表示と編集
- localStorage、ローカルJSON、将来のBlob Storage保存

## 起動

```bash
node server/dev-server.js
```

ブラウザで開く:

```txt
http://127.0.0.1:3000
```

## チェック

```bash
node --check server/dev-server.js
node --check api/handlers/plans.js
node --check api/lib/storage.local.js
node --check web/app.js
```

`npm` がある環境では `npm run dev` と `npm run check` でも実行できます。

## GitHub Pages

`docs/` 配下に静的GUIを配置しています。GitHub PagesではAPIは動かないため、保存はブラウザのlocalStorageのみです。
画面の本体は `web/` です。GitHub Pagesへ反映する前に、次のコマンドで `web/` の内容を `docs/` へ同期します。

```bash
npm run sync-docs
```

公開URL:

```txt
https://nomhck.github.io/e_relations/
```

注意:

- `server/` と `api/` はGitHub Pagesでは動きません。
- 複数人で同じ工程を保存・共有する場合はAzure Static Web Apps + APIへ移行します。

## Security Notes

- このリポジトリには秘密情報、APIキー、GitHub token、Azure接続文字列を入れない
- `.env`, `.env.*`, `local.settings.json`, `data/plans/*.json` はgit管理対象外です。
- ローカル開発サーバーは既定で `127.0.0.1` のみにbindします。
- GitHub Pages版は公開UIデモです。業務上の実データや機密工程を埋め込まないでください。
- `server/` のAPIはMVP用です。インターネット公開する場合は認証・編集キー・保存データ検証を追加してください。

# Relation Tool v2.1 改修仕様（バックログ一括）

対象：`index_dev_a_patched.html`（GitHub Pages 公開版 e_relations）
目的：安全性 → 認知負荷 → 分析価値 → 仕上げ、の順に段階的改善

---

## 🔴 Phase 1：安全性の確保（1〜2週）

### P0-1. Undo/Redo ＋ オートスナップショット
- **コマンドスタック方式**で以下を記録：NODE/EDGE/ZONE の ADD/UPDATE/DELETE、LEVEL_SET、BULK_DELETE_EDGES
- 上限 **50手**、差分のみ保持、`localStorage: rt_undo_stack` に永続化
- ショートカット：`Ctrl/Cmd+Z` / `Ctrl/Cmd+Shift+Z`
- ツールバーに `↶ 元に戻す` `↷ やり直し` を追加（disabled 状態も明示）
- **オートスナップショット**：5分ごと／±10% 変化時／破壊操作直前に自動保存、最大5世代ローテ
- キー命名 `rt_snap_YYYYMMDD_HHmmss`、`⚙ 設定 ▸ スナップショット履歴` から復元
- localStorage 使用量が 4MB 超で古い順に自動削除

### P0-2. ツールバー再構造化＋破壊操作の隔離
- ツールバーを **[編集モード] | [フィルタ] | [レイアウト] | [履歴] | [I/O] | [ビュー] | [危険操作]** にセグメント化
- グループ間に縦セパレータ、上部にマイクロラベル（10px, uppercase）
- 幅 1200px 未満で **2段折り返し**（横スクロール禁止）
- **🗑 リレーション全削除** は `︙` オーバーフローメニュー配下へ隔離、色 `#c0392b`
- 削除操作は **2段階確認**：Step1で件数明示、Step2で `DELETE` タイプ入力必須
- ダイアログ表示前に自動スナップショット取得、「復元可能」バッジ表示
- 全削除到達までのクリック数：1→**3クリック**

---

## 🟠 Phase 2：認知負荷の削減（2〜3週）

### P1-1. モードバッジ ＋ Esc 統一
- キャンバス右上に **`MODE: {現在のモード}`** バッジを常設（`top:80px; right:24px`）
- 色分け：接続=青／レベル=紫／ハイライト=橙／分析=緑／通常=灰
- バッジ下に **1行ヒント**（例：接続 →「ノードをクリック→ドラッグで接続」）
- **`Esc` で必ず通常モードへ**（ダイアログが開いていればダイアログ閉じが優先）
- モードボタンは **ラジオボタン相当**、同時 active は1つのみ
- 状態管理を `appState.mode` に単一集約するリファクタ

### P1-2. タスク詳細ダイアログ再設計
- セクション構成：
  - **基本情報**：タスク名（必須）／説明／所属ゾーン
  - **スケジュール**：`計画開始・計画終了 / 実績開始・実績終了` を 2×2 グリッド化
  - **ステータス**：自動判定と手動指定を **排他化**（日付条件を満たせば自動、満たさなければ手動）
  - **詳細 ▾**（デフォルト折り畳み）：DB ID／リレーション情報
- 判定ルール：
  | 状態 | 条件 |
  |---|---|
  | 未着手 | 実績開始なし & 計画開始 ≤ 今日 |
  | 進行中 | 実績開始あり & 実績終了なし |
  | 完了 | 実績開始あり & 実績終了あり |
  | 遅延 | 計画終了 < 今日 & 実績終了なし |
- 自動判定の**根拠を常時1行表示**
- 「詳細 ▾」の開閉状態はセッション記憶
- バリデーション：計画開始 > 計画終了 は保存不可（赤字）、実績終了 > 今日 は黄色警告

### P1-3. サイドバーのタブ化
- 通常時：`タスク | リレーション | ゾーン | 分析` の4タブ
- 接続モード時：上部に「接続ペア」パネルを**スタック固定表示**
- レベル指定モード時：上部に「対象レベル選択」パネルを固定表示
- URL ハッシュ（`#tab=tasks`）で状態保持、リロード復帰対応
- タブ内スクロール位置はセッション記憶

## 🟡 Phase 3：分析価値の強化（2〜3週）

### P2-1. アイコン刷新（絵文字→SVG＋ラベル）
- Lucide SVG（MIT）に統一、絵文字を全廃
- 全ツールバーボタンで **アイコン＋短ラベル併記**（例：`<link> 接続` `<grid> 整列`）
- 全ボタンに `aria-label` 必須、SVG に `role="img"` と `<title>`
- CSS 変数 `--icon-primary` `--icon-danger` `--icon-muted` を定義、ダークモード自動反転
- Win Chrome / macOS Safari / iPad Safari の3環境で見た目を統一

### P2-2. 情報種類フィルタ ＋ 凡例常設
- 情報種類マスターを拡張：`{ id, label, color, short, desc }`
- キャンバス右下に **凡例パネル**を常設（折り畳み可）
- 各エントリクリックで **フィルタ ON/OFF**（複数選択可）
- エッジ描画に自動反映：承認=実線赤／情報連携=破線青／承認待ち=点線橙 など
- 非該当エッジは **透明度30%** に減光
- フィルタ状態は localStorage で永続化
- CSV/XLSX エクスポート時に情報種類も含める

### P2-3. クリティカルパスのキャンバス可視化
- 有向グラフの最長経路を計算、**最長タイ全て**を保持
- 統計ダッシュボードの「CP長」隣に **`パスを表示` ボタン**
- CP を赤系（`#e74c3c`）で強調、CP外は透明度25%に減光
- 複数パスは `パス 1/3` ドロップダウンで切替
- 「CPだけ書き出し」ボタンで CP 上ノードのみエクスポート
- 500 ノードで 1秒以内、`Esc` で解除

### P2-4. 祖先〜子孫パスの一括ハイライト【新規】
- **トリガ**：通常モードで **ノードを Alt+クリック**（または右クリック→「パスを表示」）
- **解除**：`Esc` ／別ノード Alt+クリック ／空白 Alt+クリック
- **パス定義**：
  - 祖先＝選択ノードに到達できる全ノード
  - 子孫＝選択ノードから到達できる全ノード
  - デフォルト深さ：**無制限**、UI で `祖先 [∞ ▽] 子孫 [∞ ▽]`（`1/2/3/∞`）から選択可
- **描画**：
  | 対象 | 表現 |
  |---|---|
  | 選択ノード | 白フチ＋発光 |
  | 祖先 | 青系 `#3498db` |
  | 子孫 | 緑系 `#27ae60` |
  | 上流エッジ | 青3px、流向アニメ（上→下） |
  | 下流エッジ | 緑3px、流向アニメ（下→上） |
  | 対象外 | 透明度20% に減光 |
- 深さレベルごとに段階的に彩度を下げる（Lv1濃い→Lv3薄い）
- CP 表示と同時ON時は **CP を優先**（赤オーバーレイ）
- **サイドバー情報パネル**：選択ノード名／祖先件数＋直近3件／子孫件数＋直近3件／`[全祖先を選択] [全子孫を選択] [パスをCSVで書き出し]`
- **プレビュー**：Alt キー押下中のホバーで一時プレビュー、離すと戻る
- **キーボード**：`↑` = 上流へ移動／`↓` = 下流へ移動（分岐時は次数の高い方優先）
- **性能**：BFS 使用、800ノードで50ms 以内、5000ノードで300ms 以内

## 🟢 Phase 4：仕上げ（数日）

### P3-1. サーバー保存アップグレード導線
- サイドバー下部に常設バッジ `💾 保存先：このブラウザのみ`
- クリックで説明モーダル：現在のデータ量／サーバー版のメリット／`エクスポートしてサーバー版に持ち込む` ボタン
- ZIP（`nodes.csv + edges.csv + zones.csv + master.json`）で一括DL

### P3-2. ショートカット一覧（`?` パネル）
- `?` または `Shift+/` でオーバーレイ表示、`Esc` で閉じる
- 4カラム：
  - 全般：`?` `Esc` `Ctrl+Z` `Ctrl+Shift+Z`
  - モード：`C`=接続 / `L`=レベル / `H`=ハイライト / `A`=分析
  - ビュー：`1`=通常 / `2`=3D / `3`=断面 / `0`=全体表示
  - I/O：`Ctrl+S`=エクスポート / `Ctrl+O`=インポート
- ダイアログ表示中は `?` を無効化
