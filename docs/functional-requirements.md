# E Relations Functional Requirements

この文書は、EPC工程の依存関係をGUIで編集するサービスの機能要件をまとめる。
環境要件は現状維持とし、ローカルではNode標準機能のみ、公開デモはGitHub Pages、本番候補はAzure Static Web Apps + Functions + Blob Storageを前提にする。

## 採用方針

添付仕様のRelation Tool v2.1は高機能だが、初期版では実装を軽く保つ。
そのため、工程管理として必須の「タスク」「依存線」「CPM計算」「複数ビュー」「編集保存」を優先し、3Dビューや高度分析などは後続候補に分ける。

## 対象ユーザー

- EPC工事、プラント建設、設備更新などの工程担当者
- Engineering / Procurement / Fabrication / Construction / Commissioning の関係を見ながら調整する人
- Excelや工程表だけでは見えにくい依存関係、クリティカル、余裕日数をGUIで確認したい人

## データモデル

### Task

初期版で扱うタスク項目:

| 項目 | 内容 | 初期版 |
|---|---|---|
| `id` | 内部ID | 必須 |
| `code` | タスクコード | 必須 |
| `name` | タスク名 | 必須 |
| `area` | 工程領域 | 必須 |
| `owner` | 担当 | 任意 |
| `duration` | 期間（日） | 必須 |
| `progress` | 進捗率 0-100 | 任意 |
| `x`, `y` | ネットワーク図の配置座標 | 必須 |

次段階で追加するタスク項目:

| 項目 | 内容 |
|---|---|
| `description` | 説明、メモ |
| `status` | 未着手、進行中、完了 |
| `plannedStart`, `plannedEnd` | 予定開始、予定終了 |
| `actualStart`, `actualEnd` | 実績開始、実績終了 |
| `level` | 重要度、階層、粒度 |
| `affiliationId` | 所属マスター参照 |

### Dependency

初期版で扱う依存線項目:

| 項目 | 内容 | 初期版 |
|---|---|---|
| `id` | 内部ID | 必須 |
| `from` | 先行タスクID | 必須 |
| `to` | 後続タスクID | 必須 |
| `type` | FS / SS / FF / SF | 必須 |
| `lag` | ラグ日数 | 必須 |

次段階で追加する依存線項目:

| 項目 | 内容 |
|---|---|
| `label` | 情報種類、成果物、制約理由などの補足ラベル |

## 必須機能

### FR-TASK: タスク管理

| ID | 要件 |
|---|---|
| FR-TASK-001 | タスクを追加できる |
| FR-TASK-002 | タスクコード、タスク名、領域、担当、期間、進捗を編集できる |
| FR-TASK-003 | ネットワーク図上でタスクをドラッグ配置できる |
| FR-TASK-004 | 表形式ビューから複数タスクを連続編集できる |
| FR-TASK-005 | 選択中タスクを詳細パネルで編集できる |
| FR-TASK-006 | サンプルEPC工程に初期化できる |
| FR-TASK-007 | 説明、ステータス、予定日、実績日、レベルを編集できる |

### FR-DEP: 依存関係管理

| ID | 要件 |
|---|---|
| FR-DEP-001 | 接続モードでタスク間に依存線を作成できる |
| FR-DEP-002 | FS / SS / FF / SF の依存種別を指定できる |
| FR-DEP-003 | 依存線ごとにラグ日数を指定できる |
| FR-DEP-004 | 同一タスク間の重複依存は更新扱いにする |
| FR-DEP-005 | 自己依存は禁止する |
| FR-DEP-006 | 循環依存は禁止する |
| FR-DEP-007 | 依存線は視認しやすいように端点をずらして描画する |
| FR-DEP-008 | 依存線ラベルを表示、編集できる |

### FR-VIEW: 表示

| ID | 要件 |
|---|---|
| FR-VIEW-001 | ネットワーク図でタスクと依存線を表示する |
| FR-VIEW-002 | ガントビューで計算開始日、期間、クリティカルを表示する |
| FR-VIEW-003 | 表ビューでタスク一覧、先行タスク、余裕日数を表示する |
| FR-VIEW-004 | 選択タスクは各ビューで分かるように強調する |
| FR-VIEW-005 | クリティカルタスクとクリティカル依存線を強調する |

### FR-SCH: スケジュール計算

| ID | 要件 |
|---|---|
| FR-SCH-001 | DAGとして依存関係を扱う |
| FR-SCH-002 | トポロジカル順序で最早開始、最早終了を計算する |
| FR-SCH-003 | 後ろ向き計算で最遅開始、最遅終了、余裕日数を計算する |
| FR-SCH-004 | 余裕日数が0以下のタスクをクリティカルとして扱う |
| FR-SCH-005 | FS / SS / FF / SF とラグをCPM計算に反映する |

### FR-FILTER: 検索と絞り込み

| ID | 要件 |
|---|---|
| FR-FILTER-001 | 領域別にタスクを絞り込める |
| FR-FILTER-002 | コード、タスク名、担当、領域で検索できる |
| FR-FILTER-003 | クリティカルまたは低余裕日数のタスクをフォーカス一覧に表示する |
| FR-FILTER-004 | 所属/領域マスターを追加し、タスク編集候補に反映できる |

### FR-SAVE: 保存

| ID | 要件 |
|---|---|
| FR-SAVE-001 | ブラウザのlocalStorageに自動保存する |
| FR-SAVE-002 | ローカル開発サーバー起動時はJSONファイルにも保存する |
| FR-SAVE-003 | GitHub PagesではAPIなしでlocalStorage保存のみで動く |
| FR-SAVE-004 | Azure移行時はBlob Storageに工程JSONを保存する |
| FR-SAVE-005 | Azure移行時は編集キーをハッシュ化して保存する |
| FR-SAVE-006 | Azure移行時はETagで同時保存衝突を検出する |

## 次段階の機能

| ID | 要件 | 優先度 |
|---|---|---|
| FR-NEXT-004 | CSVインポート、CSVエクスポートを追加する | 中 |
| FR-NEXT-005 | 変更履歴をJSONとして保存する | 中 |
| FR-NEXT-006 | 読み取り専用URLと編集URLを明確に分ける | 中 |
| FR-NEXT-007 | 遅延開始、遅延終了などの日程ステータスを自動判定する | 中 |
| FR-NEXT-008 | ボトルネック、中心性、世代、連結成分などの分析を追加する | 低 |
| FR-NEXT-009 | サブグラフ、断面ビュー、3Dビューを追加する | 低 |

## 追加実装済み機能

- タスク詳細に説明、ステータス、予定日、実績日、レベルを追加
- 表ビューにステータス、レベル、予定完了を追加
- 所属/領域マスターの追加と未使用カスタム項目の削除
- 所属/領域ごとのノード色分け
- 依存線ラベルの作成、表示、詳細パネルからの編集
- 予定日と現在日から開始遅延、終了遅延を表示

## 初期版では扱わないもの

- Cosmos DB、SQL、Data API BuilderなどのDB構成
- Entra IDやKey Vaultなどの重いAzure構成
- リアルタイム共同編集
- Excelライブラリ前提の複雑な取込
- 3Dビュー、断面ビュー、高度なグラフ分析の初期実装

## 検証ポイント

- タスク追加、編集、ドラッグ後にlocalStorageへ保存されること
- 表ビューの入力、セレクト操作中に選択や再描画が邪魔しないこと
- 依存線作成時に循環依存が拒否されること
- FS / SS / FF / SF とラグ変更でガントと余裕日数が更新されること
- 説明、ステータス、予定日、実績日、レベル、依存線ラベルが保存されること
- 所属/領域の追加がフィルター、表ビュー、詳細パネルに反映されること
- GitHub PagesではAPIエラーになっても画面操作が継続できること
- ローカル開発サーバーでは `data/plans/*.json` に保存されること
